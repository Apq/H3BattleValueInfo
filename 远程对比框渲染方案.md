# 战场远程对比框渲染方案

## 概述

远程对比框需要在战斗过程中实时更新数值（远程攻击、法术输出、总计），背景为纯色色块 + 红色数字。

## 渲染架构

### 三步渲染流程

```
1. 合成 bg_（内存中的 RGB565 Pcx16 对象）
      - 纯色填充
      - 文字 TextDraw 叠加到 bg_
      ↓
2. 一次 Blt 到 DirectDraw backbuffer
      - DrawPcx16ToBackBuffer(bg_, x, y, TRUE)
      - TRUE = 跳过透明像素（保留边框等）
      ↓
3. H3Redraw 强制 GPU 刷新该区域
      - o_WndMgr->H3Redraw(x, y, panel_w, panel_h)
      - 让 GPU 重新从 backbuffer 上传显示
```

### 为什么需要三步

- **bg_ 合成**：避免每次重绘都改 screen_pcx16（HD Mod 中 screen_pcx16 只是 Blt 来源，不是最终显示）
- **backbuffer blt**：backbuffer 是最终显示表面，英雄悬浮框也用它
- **H3Redraw**：没有这步，GPU 不会重新读取我们刚写入的 backbuffer 区域（HD Mod 的 OpenGL 层不会自动刷新）

## 关键函数

### DrawPcx16ToBackBuffer

将 `_Pcx16_` 对象（RGB565）blt 到 `o_DDSurfaceBackBuffer`（游戏的主 DirectDraw surface）。

```cpp
// 原型
static bool DrawPcx16ToBackBuffer(_Pcx16_* src, int dst_x, int dst_y, bool skip_sentinel);

// 用法：透明模式，把 bg_ 画到 backbuffer 的 (x, y) 位置
DrawPcx16ToBackBuffer(bg_, x, y, TRUE);
```

- `skip_sentinel=TRUE`：只复制非透明像素（用 `TEXT_MASK_SENTINEL_565` 标记透明区域）
- 自动处理 16bpp/32bpp backbuffer 格式转换

### H3Redraw

强制游戏刷新指定区域。内部会触发 `Hook_AfterBlt`，但因为重绘的是面板区域（非战斗对话框），不会递归。

```cpp
// 原型（定义在 Compat.inc.cpp）
struct _WndMgr_ {
    void H3Redraw(INT32 x, INT32 y, INT32 dx, INT32 dy) {
        THISCALL_5(VOID, 0x603190, this, x, y, dx, dy);
    }
};

// 用法
o_WndMgr->H3Redraw(panel_x, panel_y, panel_w, panel_h);
```

## 渲染时机

Hook 地址：`0x495C50`（`Hook_CycleCombatScreen`），HiHook，在原函数调用**之后**执行。

- 此时对话框已渲染完毕，backbuffer 包含最新画面
- 在这个时机画面板 → blt 到 backbuffer → H3Redraw

## 文字绘制

文字用 H3 原生 `H3Font::TextDraw` 画到 `bg_`（RGB565），再用 `DrawPcx16ToBackBuffer` 一次性 blt 到 backbuffer。

这样做的好处：
- 文字和背景在同一次 blt 中合成，避免分两次 blt 导致的叠加问题
- `bg_` 每次 DrawImpl 调用时重新合成，保证文字是最新的

```cpp
// 1. 先清空文字区域（填充 sentinel）
for (int row = 2; row < panel_h - 2; ++row) {
    _word_* p = (_word_*)(bg_->buffer + row * bg_->scanlineSize);
    for (int col = pad_x; col < panel_w - pad_x; ++col)
        p[col] = TEXT_MASK_SENTINEL_565;
}

// 2. 文字画到 bg_
_Fnt_* font = GetRangedPanelTextFont();
if (font) {
    char tmp[64];
    _snprintf(tmp, sizeof(tmp) - 1, "%d", ranged_value_[0]);
    font->TextDraw(bg_, tmp, pad_x, 2, col_w, 18,
        NH3Dlg::eTextColor::RED,
        static_cast<NH3Dlg::eTextAlignment>(4));
    // ... 其他行同理
}

// 3. 背景+文字一起 blt 到 backbuffer
DrawPcx16ToBackBuffer(bg_, x, y, TRUE);

// 4. 强制刷新
o_WndMgr->H3Redraw(x, y, panel_w, panel_h);
```

## 位置

面板放在**战场对话框外侧上方**：

```cpp
int x = battle_x + (800 - panel_w) / 2;
int y = battle_y - panel_h;  // dlg->y - panel_h
```

## 诊断历史

- **screen_pcx16 方案**：文字首帧显示，后续不更新（HD Mod 中 screen_pcx16 只是 Blt 来源，不是 GPU 最终读取的表面）
- **backbuffer 无 H3Redraw**：同样首帧显示，后续不更新（OpenGL 层缓存了旧内容）
- **backbuffer + H3Redraw**：正常工作 ✅
