| 软件 | 版本 | 说明 |
|------|------|------|

项目骨架已就绪，**无需手动创建项目**：

## 3. vcxproj 关键配置说明

> 以下设置已写入 vcxproj，无需手动在 VS 属性页操作。

| 配置项 | 值 | 原因 |
|--------|-----|------|
| 平台 | **Win32 (x86)** | 游戏是 32 位进程；x64 DLL 无法注入 |
| 结构成员对齐 `/Zp1` | 1 字节 | `homm3.h` 要求，否则所有结构体偏移错误 |
| 预处理宏 `WINDOWS_IGNORE_PACKING_MISMATCH` | 已定义 | `/Zp1` 与 Windows SDK 静态断言冲突，需此宏消除 |
| 预处理宏 `_CRT_SECURE_NO_WARNINGS` | 已定义 | 消除 HotA 库中 `vsprintf`/`sscanf` 的弃用警告 |
| 编码 `/utf-8` | 是 | 源文件 UTF-8，中文不乱码 |
| 预编译头 | 关闭 | HotA 库不兼容 PCH |
| 符合模式 | 关闭 | HotA 库使用旧式 C++ 写法 |
| UAC | 关闭 | DLL 注入不需要 UAC |

## 4. 首次搭建步骤（仅在全新克隆后执行）

### 4.1 复制 deps 文件

> `xcopy` 无法处理路径中的中文，必须用 PowerShell。

创建并执行以下脚本（执行后删除）：

```powershell
$base = 'd:\ApqGit\heroes3\sp'
$hotaDir = Get-ChildItem $base -Directory | Where-Object { $_.Name -like 'HotA--c*' } | Select-Object -First 1
$dst = Join-Path $base 'BattleValueInfo\deps'
New-Item -ItemType Directory -Force $dst | Out-Null
Get-ChildItem (Join-Path $hotaDir.FullName 'HotA') | Copy-Item -Destination $dst -Force
```

### 4.2 转换 deps 文件编码（GBK → UTF-8）

> HotA 库所有文件为 GBK 编码，`/utf-8` 编译参数会触发 C4828 警告，必须转码。

```powershell
$gbk = [System.Text.Encoding]::GetEncoding(936)
$utf8 = [System.Text.UTF8Encoding]::new($false)
Get-ChildItem 'd:\ApqGit\heroes3\sp\BattleValueInfo\deps' -Include *.h,*.hpp,*.cpp -Recurse | ForEach-Object {
    $text = $gbk.GetString([System.IO.File]::ReadAllBytes($_.FullName))
    [System.IO.File]::WriteAllText($_.FullName, $text, $utf8)
}
```

## 5. 编译

1. VS 工具栏平台选 **x86**（不能用 x64）
2. `Ctrl+Shift+B`
3. 输出 DLL：`Release\BattleValueInfo.dll`

## 7. 已知警告（无害，不影响功能）

| 警告 | 来源 | 说明 |
|------|------|------|
| C4235 | `patcher_x86.hpp` | `__asm` 在结构体内，x86 MSVC 支持但非标准 |
| C4010 | `patcher_x86.hpp` | 注释中含行继续符，库自身问题 |
| C4311/C4312/C4302 | `patcher_x86.hpp` | **仅 x64 配置**：指针截断，因该库不支持 x64 |
