# BattleValueInfo 编译与部署指南

## 环境

| 项 | 当前要求 |
|---|---|
| Visual Studio | Visual Studio 18 Enterprise |
| MSBuild | `C:\Program Files\Microsoft Visual Studio\18\Enterprise\MSBuild\Current\Bin\MSBuild.exe` |
| 平台 | Win32 / x86 |
| 配置 | Release |
| C++ 标准 | C++20 |
| 结构对齐 | `/Zp1` |
| 编码 | `/utf-8` |

必须构建 Win32。游戏是 32 位进程，x64 DLL 不能加载。

## 项目结构

```text
H3BattleValueInfo/
├── BattleValueInfo.cpp
├── BattleValueInfo.vcxproj
├── BattleValueInfo.ini
├── deploy.ps1
├── deps/
├── img/
├── modules/
├── README.md
├── 设计文档.md
├── 远程输出能力算法.md
└── 背景图渲染方案.md
```

`modules/*.inc.cpp` 由 `BattleValueInfo.cpp` include。修改 `.inc.cpp` 后，如果 MSBuild 未检测到变化，可以 touch 主文件再编译：

```powershell
(Get-Item BattleValueInfo.cpp).LastWriteTime = Get-Date
```

## 关键 vcxproj 设置

| 配置项 | 值 | 原因 |
|---|---|---|
| 平台 | Win32 | 游戏进程是 x86 |
| 结构成员对齐 | `/Zp1` | HoMM3/H3API 裸偏移要求 |
| `WINDOWS_IGNORE_PACKING_MISMATCH` | 已定义 | 避免 Windows SDK 与 `/Zp1` 的静态断言冲突 |
| `_CRT_SECURE_NO_WARNINGS` | 已定义 | 允许旧式 CRT 调用 |
| `/utf-8` | 已启用 | 源文件和中文日志使用 UTF-8 |
| 预编译头 | 关闭 | 兼容 H3API/patcher_x86 |
| UAC | 关闭 | DLL 注入不需要 UAC |

## 编译

在项目目录执行：

```powershell
& "C:\Program Files\Microsoft Visual Studio\18\Enterprise\MSBuild\Current\Bin\MSBuild.exe" BattleValueInfo.vcxproj /p:Configuration=Release /p:Platform=Win32 /m
```

输出：

```text
Release\BattleValueInfo.dll
```

成功输出示例：

```text
已成功生成。
0 个警告
0 个错误
```

## 部署

默认部署脚本：

```powershell
.\deploy.ps1
```

当前脚本部署到：

```text
D:\Heroes3\Heroes3_2026.05.01\_HD3_Data\Packs\战斗价值信息
```

部署内容：

1. `Release\BattleValueInfo.dll`
2. `BattleValueInfo.ini`
3. `img\*.*`

脚本会提示：

```text
BattleValueInfo 依赖 MegaDesc，请先部署并启用 MegaDesc。
```

## 校验 DLL

部署后可校验源 DLL 和目标 DLL 是否一致：

```powershell
Get-FileHash Release\BattleValueInfo.dll -Algorithm MD5
Get-FileHash "D:\Heroes3\Heroes3_2026.05.01\_HD3_Data\Packs\战斗价值信息\BattleValueInfo.dll" -Algorithm MD5
```

两个 MD5 应一致。

## 配置文件

配置文件为 [BattleValueInfo.ini](BattleValueInfo.ini)。修改后需要重启游戏。

`[RangedPanel] Y=0` 是推荐默认值。代码内部已有 `RP_BASE_Y_OFFSET=23`，配置项只用于微调。

## 日志

默认生成在部署目录：

```text
BattleValueInfo_YYYYMMDD_HHMMSS.log
```

`DisableLog=1` 可关闭日志。

正常情况下日志只记录加载/启用信息；异常或资源失败时再记录故障细节。常见关键字：

- `BattleValueInfo 正在加载`
- `BattleValueInfo 已启用`
- `SEH exception`
- `[DDBackground] ... failed`

## 常见问题

### 修改 `.inc.cpp` 后显示“所有输出均为最新”

Touch 主文件后重新编译：

```powershell
(Get-Item BattleValueInfo.cpp).LastWriteTime = Get-Date
```

### 面板不显示

检查：

1. 是否进入真正手动战斗。
2. 是否处于快速战斗结果界面或战斗结果界面。
3. `BattleValueInfo.ini` 中 `[RangedPanel]` 的 `Width`、`Height`、`Y` 是否合理。
4. 日志中是否有 `SEH exception` 或 `[DDBackground] ... failed`。

### 远程或魔法值异常

优先检查结构偏移是否被改坏：

- `_BattleStack_` 必须为 `0x548`。
- `_Spell_` 必须为 `0x88`。
- `_Artifact_` 必须为 8 字节。
- 魔法书槽位必须是 `bodyArtifacts[17]`。
- `CanShoot` 地址必须是 `0x442610`。
