# BattleValueInfo 设计文档

本文描述当前实现。窗口扩展、背景、按钮和描述区布局由 MegaDesc 负责，BattleValueInfo 只补充战斗价值文本和战斗顶部输出对比面板。

## 功能范围

BattleValueInfo 负责：

1. 在生物信息窗口中追加“战斗价值”行。
2. 在手动战斗中绘制远程/魔法/总输出对比面板。
3. 快速战斗结果界面、战斗结果界面弹出时隐藏对比面板。
4. 点击取消重新手动战斗后恢复对比面板。

BattleValueInfo 不负责：

1. 生物信息窗口加高。
2. 生物信息窗口背景替换。
3. 原窗口按钮替换。
4. 描述区扩展和窗口位置吸附。

这些由 MegaDesc 项目处理。

## 生物战斗价值

显示值来源：

```text
*(int*)(*(char**)0x6747B0 + creature_id * 0x74 + 0x3C)
```

含义：

- 全局生物表指针：`0x6747B0`
- `H3CreatureInformation` 大小：`0x74`
- `fightValue` 偏移：`0x3C`

主显示值是单只生物的 `fight_value`，与数量无关。

战斗中如果能匹配当前 `_BattleStack_`，括号内追加队首单只按剩余 HP 折算后的值：

```text
括号值 = fight_value * (hit_points - lost_hp) / hit_points
```

新增控件：

| 控件 | ID |
|---|---:|
| label | 3009 |
| value | 3008 |

战斗价值行 Y 坐标：

```text
name_y + FightValueYOffset
```

默认 `FightValueYOffset=8`。

## 战斗输出面板

面板不是 HoMM3 `_Dlg_` 或 `_DlgItem_`，不接收输入，不改变当前窗口。它在 `BattleRedraw @ 0x493FC0` 后绘制到 DirectDraw backbuffer。

面板锚点：

```text
battle_x = (screen_width - 800) / 2
battle_y = (screen_height - 600) / 2
x = battle_x + (800 - panel_width) / 2
y = battle_y - panel_height - RP_BASE_Y_OFFSET - cfg.Y
```

其中：

- `RP_BASE_Y_OFFSET=23`
- `cfg.Y` 默认为 `0`，只用于微调
- `cfg.Y` 越大，面板越往上
- 面板位于 800x600 战场框外部上方

面板三行含义：

| 行 | 含义 |
|---|---|
| 远程输出 | 本方所有可射击远程 stack 对敌方各存活 stack 逐一试算后的最小输出 |
| 魔法输出 | 本方英雄白名单攻击魔法中的最高理论单次伤害 |
| 总输出 | 远程输出 + 魔法输出 |

详细算法见 [远程输出能力算法.md](远程输出能力算法.md)。

## 面板生命周期

面板只在真正进入手动战斗后显示。

流程：

1. `Combat_StartBattle @ 0x4781C0` 后调用 `BeginManualBattle()`，面板 armed。
2. `BattleRedraw @ 0x493FC0` 后调用 `Draw()`。
3. 快速战斗结果界面或战斗结果界面出现时调用 `SuppressForResult()`，面板隐藏。
4. 从结果界面点击取消重打后，结果窗口消失，面板重新 armed。
5. 战斗结束/销毁时 `Destroy()` 清理资源。

快速战斗结果窗口识别：HD Mod Replayable Quick Battle 的取消/重打按钮 ID 为 `0x1FB / 507`。

另外还使用原版状态函数辅助判断：

| 地址 | 用途 |
|---:|---|
| `0x465410` | `IsBattleOver` |
| `0x46A080` | `IsHiddenBattle` |

## 面板残留处理

面板画在战场框外部上方，涉及两个输出目标：DD backbuffer（背景）和 `screen_pcx16`（文字）。残留也分两类。

### DD backbuffer 残留

当前实现：

1. 每次画面板前，把面板区域底下的 backbuffer 原始像素复制到离屏 DirectDraw surface。
2. suppress/destroy 时，把保存的原始像素 Blt 回 backbuffer。
3. 避免黑块，也避免面板背景残留。

### screen_pcx16 残留

`screen_pcx16` 不会自动清除面板区域的旧文字，每帧画面板背景后、画文字前，用 `ClearScreenRegion()` 将面板区域填 0。

相关函数：

| 函数 | 用途 |
|---|---|
| `CreateEmptyDDSurface()` | 创建空离屏 surface |
| `CopyBackBufferToSurface()` | 保存面板底下原始像素 |
| `RestoreSurfaceToBackBuffer()` | 恢复原始像素 |
| `SaveUnderlay()` | 面板绘制前保存 underlay |
| `RestoreUnderlay()` | 面板隐藏/销毁时恢复 underlay |
| `ClearScreenRegion()` | 清除 screen_pcx16 面板区域旧文字 |

## Hook 列表

| Hook | 地址 | 类型 | 说明 |
|---|---:|---|---|
| BuildCombat | `0x5F4503` | LoHook | 战斗生物信息窗口构建后插入战斗价值 |
| BuildAdventure | `0x5F3E75` | LoHook | 冒险生物信息窗口构建后插入战斗价值 |
| BuildTown | `0x5F491E` | LoHook | 城镇生物信息窗口构建后插入战斗价值 |
| DlgDefProc | `0x41B120` | HiHook | 兜底补插战斗价值 |
| BattleRedraw | `0x493FC0` | HiHook | 战场重绘后刷新输出面板 |
| CombatStartBattle | `0x4781C0` | HiHook | 真正进入手动战斗后 armed 面板 |
| CastSpell | `0x464F10` | HiHook | 施法后标记面板 dirty |

当前没有注册 `0x4746B0` 行动 hook；栈变化主要通过 checksum 自动发现。

## 配置

见 [BattleValueInfo.ini](BattleValueInfo.ini)。当前默认：

```ini
[RangedPanel]
BackgroundImage=rp_bg.pcx
TextFont=smalfont.fnt
TextColor=4
Width=298
Height=93
Y=0
Row1Y=24
Row2Y=42
Row3Y=60
```

## 日志

日志默认启用，保留最近 30 个：

```ini
[Logging]
DisableLog=0
```

部署目录下生成：

```text
BattleValueInfo_YYYYMMDD_HHMMSS.log
```

常见诊断关键字：

- `[RangedOverlayPanel] armed`
- `[RangedOverlayPanel] suppressed`
- `[RangedOverlayPanel] recalculated`
- `[StackDiag]`
- `[RangedCalc]`
- `SEH exception`

## 依赖和部署

BattleValueInfo 与 MegaDesc 是独立项目，各自编译和部署。BattleValueInfo 需要 MegaDesc 先部署并在 HD Launcher 中启用。

部署脚本：[deploy.ps1](deploy.ps1)。
