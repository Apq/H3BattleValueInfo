# 远优对比设计文档

本文描述当前实现。生物信息窗口扩展由 MegaDesc 负责，远优对比只补充战斗价值文本和战斗顶部输出对比面板。

## 功能范围

远优对比负责：

1. 在生物信息窗口中追加"战斗价值"行。
2. 在手动战斗中绘制远程/魔法/总输出对比面板。
3. 快速战斗结果界面、战斗结果界面弹出时隐藏对比面板。
4. 点击取消重新手动战斗后恢复对比面板。

远优对比不负责：

1. 生物信息窗口加高。
2. 生物信息窗口背景替换。
3. 原窗口按钮替换。
4. 描述区扩展和窗口位置吸附。

这些由 MegaDesc 项目处理。

## 生物战斗价值

显示值来源：

```text
*(int*)(*(char**)0x6747B0 + creature_id * 0x74 + 0x3C)
```

含义：

- 全局生物表指针：`0x6747B0`
- `H3CreatureInformation` 大小：`0x74`
- `fightValue` 偏移：`0x3C`

主显示值是单只生物的 `fight_value`，与数量无关。

战斗中如果能匹配当前 `_BattleStack_`，括号内追加队首单只按剩余 HP 折算后的值：

```text
括号值 = fight_value * (hit_points - lost_hp) / hit_points
```

新增控件：

| 控件 | ID |
|---|---:|
| label | 3009 |
| value | 3008 |

战斗价值行 Y 坐标：`name_y + FightValueYOffset`，默认 `FightValueYOffset=8`。

## 对比面板渲染

面板不是 HoMM3 `_Dlg_` 或 `_DlgItem_`，不接收输入，不改变当前窗口。

### 当前方案：完全不用 DirectDraw

HD 版的 backbuffer 由 OpenGL 管理，不能 Lock/Blt。采用以下方案：

1. 背景图解码为 RGB565 `_Pcx16_`（`bg_`）
2. 每帧创建临时合成图（`bg_composite_`）
3. 背景复制到合成图
4. 文字叠加到合成图
5. `DrawPcx16ToBackBuffer()` 一次 blt 到 backbuffer
6. `H3Redraw()` 强制 GPU 刷新面板区域

```
PCX (8bpp indexed)
    ↓ 解码 RGB565
_Pcx16_ bg_ (背景缓存)
    ↓ 复制
_Pcx16_ bg_composite_ (每帧重建)
    ↓ TextDraw() 叠加文字
bg_composite_ (背景+文字)
    ↓ DrawPcx16ToBackBuffer() blt
backbuffer
    ↓ H3Redraw() 刷新
屏幕显示
```

### 面板布局

面板 298×93 像素，中线在 x=149。两列数字分列中线两侧：

| 列 | x | 宽度 | 对齐 | 含义 |
|---|---|---|---|---|
| 左 | 8 | 141 | MIDDLE_RIGHT | 敌方数值 |
| 右 | 154 | 144 | MIDDLE_LEFT | 本方数值 |

Y 坐标从 INI 读取：`Row1Y=24`, `Row2Y=42`, `Row3Y=60`。

三行含义：远程输出、魔法输出、总输出。

### 关键函数

| 函数 | 用途 |
|---|---|
| `CreatePcx16Background()` | 解码 PCX 为 RGB565 `_Pcx16_` |
| `EnsureBackground()` | 懒加载背景 |
| `DrawPcx16ToBackBuffer()` | blt `_Pcx16_` 到 backbuffer |
| `H3Redraw()` | 强制 GPU 刷新区域 |

## 面板生命周期

面板只在真正进入手动战斗后显示。

流程：

1. `Combat_StartBattle @ 0x4781C0` 后调用 `BeginManualBattle()`，面板 armed。
2. `Hook_CycleCombatScreen @ 0x495C50` 每帧调用 `Draw()`。
3. 快速战斗结果界面或战斗结果界面出现时调用 `SuppressForResult()`，面板隐藏。
4. 从结果界面点击取消重打后，结果窗口消失，面板重新 armed。
5. 战斗结束/销毁时 `Destroy()` 清理资源。

快速战斗结果窗口识别：HD Mod Replayable Quick Battle 的取消/重打按钮 ID 为 `0x1FB / 507`。

原版状态函数辅助判断：

| 地址 | 用途 |
|---:|---|
| `0x465410` | `IsBattleOver` |
| `0x46A080` | `IsHiddenBattle` |

## 配置

见 [BattleValueInfo.ini](BattleValueInfo.ini)。常用项：

```ini
[RangedPanel]
BackgroundImage=rp_bg.pcx
TextFont=smalfont.fnt
TextColor=4
Width=298
Height=93
Y=0
Row1Y=24
Row2Y=42
Row3Y=60

[Format]
LabelFightValue=战斗价值
```

`[RangedPanel] Y=0` 是推荐默认值。代码内部有 `RP_BASE_Y_OFFSET=23` 基础上移，配置项只用于微调；数值越大，面板越往上。
