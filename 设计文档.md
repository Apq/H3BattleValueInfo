# BattleValueInfo 设计文档

> 本文描述当前实现。窗口扩展、背景、按钮和描述区布局已迁移到 MegaDesc。

## 1. 功能范围

BattleValueInfo 只负责：

- 在 MegaDesc 已加载时，在生物信息窗口中追加生物配置值行；
- 在战斗中顶部居中显示双方远程力量面板。

不再负责：

- 生物信息窗口加高；
- 背景替换；
- 按钮替换；
- 描述区扩展和窗口位置吸附。

这些功能由 MegaDesc 项目负责。

## 2. 生物配置值显示

显示值来源：

```text
*(int*)(*(char**)0x6747B0 + creature_id * 0x74 + 0x3C)
```

其中：

- 全局生物表指针：`0x6747B0`
- `_CreatureInfo_` 结构大小：`0x74`
- `fight_value` 偏移：`+0x3C`

显示主值：

```text
生物配置值 = 单只 fight_value
```

该主值与当前数量无关。

战斗中如果能匹配到当前 `_BattleStack_`，会额外在小括号中显示队首单只按剩余 HP 折算后的值：

```text
括号值 = fight_value × (hit_points - lost_hp) / hit_points
```

其中 `lost_hp` 是当前队首那只生物已损失的 HP。

窗口中新增控件：

- label id：`3009`
- value id：`3008`

战斗价值行 Y 坐标：

```text
name_y + FightValueYOffset
```

默认 `FightValueYOffset=8`。如果 BattleValueInfo 先于 MegaDesc 插入该行，MegaDesc 后续整体下移会把该行一起移动。

## 3. Hook 列表

| Hook | 地址 | 类型 | 说明 |
|------|------|------|------|
| BuildCombat | 0x5F4503 | LoHook | 战斗生物信息窗口构建后插入战斗价值 |
| BuildAdventure | 0x5F3E75 | LoHook | 冒险生物信息窗口构建后插入战斗价值 |
| BuildTown | 0x5F491E | LoHook | 城镇生物信息窗口构建后插入战斗价值 |
| DlgDefProc | 0x41B120 | HiHook | 兜底补插战斗价值 |
| BattleRedraw | 0x493FC0 | HiHook | 战场重绘后刷新顶部远程力量面板 |

## 4. 配置文件

```ini
[Logging]
; 0 = 不禁止，正常写日志并保留最近 30 个（默认）
; 1 = 禁止，不生成日志，也不扫描/清理旧日志
DisableLog=0

[Layout]
FightValueYOffset=8

[RangedPanel]
BackgroundImage=rp_bg.pcx
TextFont=smalfont.fnt
TextColor=6
Width=500
Height=120
Y=8

[Format]
LabelFightValue=Fight Value
```

## 5. 项目依赖

BattleValueInfo 与 MegaDesc 是独立的解决方案，各自单独编译和部署。
