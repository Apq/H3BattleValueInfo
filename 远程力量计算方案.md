# 战场"远程输出"计算方案

> 日期：2026-07-01
> 依据：`Heroes3Src/docs/AI_RANGED_ADVANTAGE_ANALYSIS.md`（逆向确认）+ 插件现有数据结构

## 1. 目标

在战场顶部面板上显示双方"远程输出"对比值，帮助玩家快速判断哪方远程单位当前更能打出伤害。

"远程输出"是自定义概念，游戏中不存在单一变量。它不是部队价值，也不是胜率，而是近似表示：**本方所有远程单位在当前数量下，对敌方较高防御目标能打出的理论射击伤害总量**。

## 2. 可用数据（插件已确认可读）

### 2.1 `_BattleMgr_` 结构

```cpp
// o_BattleMgr 已全局可用
_BattleMgr_* mgr = o_BattleMgr;

// 双方英雄
_Hero_* hero[2];  // mgr->hero[0] = 攻方, mgr->hero[1] = 守方

// 战场 stack 数组
// mgr->stack[side][slot], side: 0/1, slot: 0..20
```

### 2.2 `_BattleStack_` 关键字段

| 偏移 | 字段 | 类型 | 用途 |
|------|------|------|------|
| +0x34 | `creature_id` | int | 生物类型 ID |
| +0x4C | `count_current` | int | 当前存活数量 |
| +0x58 | `lost_hp` | int | 队首已损失 HP |
| +0x6C | `full_hp` | int | 全队满 HP |

### 2.3 `_CreatureInfo_` 关键字段（通过 `stack.creature` 访问）

| 字段 | 用途 |
|------|------|
| `fight_value` | 生物基础战斗价值 |
| `hit_points` | 单只生物 HP |
| `attack` | 攻击力 |
| `defence` | 防御力 |
| `damage_min` / `damage_max` | 伤害范围 |
| `shots` | 弹药数（>0 即远程单位） |
| `speed` | 速度 |
| `spells_count` | 生物自带法术数 |

### 2.4 `_Hero_` 关键字段

| 字段 | 用途 |
|------|------|
| `spell_points` (+24) | 当前魔法值 |
| `spell[70]` (+1002) | 已掌握法术 |
| `spell_level[70]` (+1072) | 法术等级 |
| `attack` / `defence` / `spell_power` / `knowledge` | 英雄四维 |

## 3. 远程单位识别

**判定条件（已确认）：**

```cpp
bool isRanged = (stack.creature.shots > 0);
```

这与 AI 评估层的 `actionWindow == 0` 等价。原版用 `shots > 0` 或 attributes 含 `SHOOTING_ARMY` 标记远程单位。

## 4. 远程输出能力计算公式

### 4.1 核心定义

新的面板值不再用“对方最大防御”做一次性估算，而是定义为：

```text
本方远程输出能力 = min(本方全部远程单位对每一支敌方存活部队的总射击输出)
```

也就是：

1. 枚举敌方每一支存活 stack；
2. 把该 stack 作为目标；
3. 计算本方所有可射击远程 stack 对这个目标的理论输出总和；
4. 对所有敌方目标的输出结果取最小值。

这样得到的是偏保守的“远程输出能力”：

- 如果敌方有高防部队，输出会下降；
- 如果敌方有贴脸目标导致远程不能射击，输出会下降；
- 如果距离过远、穿城墙、障碍阻挡导致原版判定不能射或伤害降低，输出会下降；
- 如果英雄有幻影神弓 / 金弓等无近战/距离/城墙惩罚效果，则按游戏原版逻辑处理。

### 4.2 单个 shooter 对单个目标的输出

优先调用游戏原版战斗 stack 方法，不在插件里手写距离、障碍、城墙、贴脸判定：

```cpp
bool canShoot = shooter->CanShoot(target);
```

`CanShoot(target)` 由原版处理：

- 是否有弹药；
- 是否被贴脸；
- 是否有幻影神弓 / 金弓类神器导致贴脸或距离惩罚不生效；
- 目标是否被障碍/城墙影响；
- 当前战场状态下是否能射该目标。

可射击时，基础伤害也调用原版函数，不再手写平均伤害，然后再交给原版伤害修正函数：

```cpp
int baseDamage = CALL_2(int, __thiscall, 0x442E80, shooter, 0);
int fireshieldDamage = 0;
int damage = shooter->Calc_Damage_Bonuses(
    target,
    baseDamage,
    TRUE,   // IsShot
    TRUE,   // Virtual，虚拟计算，不实际造成伤害
    0,
    &fireshieldDamage
);
```

`0x442E80` 由原版处理基础伤害：生物伤害范围、当前数量、祝福/诅咒等基础伤害相关状态。`Calc_Damage_Bonuses(..., IsShot=TRUE, Virtual=TRUE)` 继续由原版处理：

- 攻防差；
- 英雄攻击/防御相关修正；
- 箭术；
- 距离惩罚；
- 城墙惩罚；
- 相关神器/技能/状态修正。

### 4.3 本方对单个敌方目标的总输出

```cpp
int CalcRangedOutputToTarget(_BattleMgr_* mgr, int side, _BattleStack_* target)
{
    int output = 0;
    for (int slot = 0; slot < 21; ++slot) {
        _BattleStack_* shooter = &mgr->stack[side][slot];
        if (shooter->count_current <= 0) continue;
        if (shooter->creature.shots <= 0) continue;
        if (!shooter->CanShoot(target)) continue;

        int baseDamage = CALL_2(int, __thiscall, 0x442E80, shooter, 0);
        int fireshieldDamage = 0;
        int damage = shooter->Calc_Damage_Bonuses(target, baseDamage, TRUE, TRUE, 0, &fireshieldDamage);
        if (damage > 0) output += damage;
    }
    return output;
}
```

### 4.4 一方远程输出能力

```cpp
int CalcSideRangedOutputAbility(_BattleMgr_* mgr, int side)
{
    int enemySide = 1 - side;
    bool hasTarget = false;
    int minOutput = INT_MAX;

    for (int slot = 0; slot < 21; ++slot) {
        _BattleStack_* target = &mgr->stack[enemySide][slot];
        if (target->count_current <= 0) continue;

        hasTarget = true;
        int output = CalcRangedOutputToTarget(mgr, side, target);
        if (output < minOutput) minOutput = output;
    }

    return hasTarget ? minOutput : 0;
}
```

该算法的含义是：**本方远程单位至少能对敌方任意一支存活部队打出多少输出**。

## 5. 第一版实现建议

第一版直接采用上面的逐目标算法：

```text
远程输出能力 = min(本方全部远程对每支敌方存活部队的可射击输出)
```

优点：

- 不需要单独维护距离公式；
- 不需要单独判断贴脸；
- 不需要单独判断城墙/障碍；
- 不需要手动列举幻影神弓、金弓等特殊修正；
- 与原版战斗伤害逻辑最接近。

注意：这是“保守输出能力”，不是“最佳输出”。如果敌方某一支部队当前很难射、很硬、或被障碍影响，本方显示值会被这支部队拉低。
## 6. 与当前插件代码的关系

当前 `CalcRangedUnitsPower()` 已改为“远程输出能力”计算：

```text
CalcRangedUnitsPower(side)
= min(CalcRangedOutputToTarget(side, 每一支敌方存活 stack))
```

其中 `CalcRangedOutputToTarget()` 会累加本方所有可射击远程 stack 对该目标的原版虚拟射击伤害：

```cpp
if (!shooter->CanShoot(target)) continue;
damage = shooter->Calc_Damage_Bonuses(target, baseDamage, TRUE, TRUE, 0, &fireshieldDamage);
```

所以现在显示的第一行不再是 `fight_value × count`，也不是单纯攻防差估算，而是更接近原版规则的保守远程输出能力。

面板三行含义建议：

| 行 | 含义 |
|----|------|
| 远程输出 | 本方远程对敌方各存活目标逐一计算后的最小输出 |
| 魔法输出 | 攻击魔法输出，占位，当前仍为 0 |
| 总输出 | 远程输出 + 魔法输出 |

## 7. 英雄技能读取

英雄副技能在 `_Hero_` 结构中已可读：

```cpp
struct _Hero_ {
    // ...
    _byte_ second_skill[28];       // +201，二级技能等级，索引为 HSS_xxx
    _byte_ second_skill_show[28];  // +229，显示状态
    _dword_ second_skill_count;    // +257
    _byte_ spell[70];         // +1002 已掌握法术
    _byte_ spell_level[70];   // +1072 法术等级
};
```

箭术技能 ID：

```cpp
#define HSS_ARCHERY 1
```

读取时必须考虑任意一方可能没有英雄：

```cpp
int archeryLevel = hero ? hero->second_skill[HSS_ARCHERY] : 0;
```

## 9. 实现优先级

1. **P0**：把面板概念改为"远程输出"，使用平均伤害、当前数量、攻防差、箭术修正
2. **P1**：敌方防御代表值取敌方各存活 stack 平均防御中的最大值
3. **P2**：加入距离/城墙修正（需要战场距离计算，复杂度高）
4. **P3**：加入英雄魔法力量相关行（法术伤害类远程输出）

## 10. 注意事项

- "远程输出"不是精确预测战斗结果，而是给玩家一个快速参考
- 远程单位被贴脸后实际输出会大幅下降（近战半伤），面板无法反映这种动态情况
- 城墙、障碍、距离等修正对实际伤害影响很大（各减半），但面板是实时变化的，频繁计算可能不稳定
- 建议面板只显示"基础远程火力"（不考虑距离/城墙），让玩家自己判断战场形势
