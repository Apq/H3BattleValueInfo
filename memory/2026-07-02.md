# 2026-07-02 开发笔记

## 魔力特长（Sorcery Specialty）修正

### 问题
面板魔法输出显示 2173（冰bolt），但魔法书显示 2995。差距 822。

### 根因
英雄特长是"魔力"（spec_type=0, spec_param=25=HSS_SORCERY），会随等级增强 Sorcery 技能效果。原代码只用了整数乘法 `damage * 115 / 100`，完全没有考虑特长增强。

### 关键数据结构
- **英雄 spec 表**：`*(_ptr_*)0x679C80` 指向 spec 表，每条 0x28 字节
  - `+0`: spec_type（0=技能特长, 3=法术特长）
  - `+4`: spec_param（技能特长时=HSS_常量，如 25=SORCERY）
- **hero_id**：`*(int*)((char*)hero + 0x1a)`
- **hero_level**：`*(short*)((char*)hero + 0x55)`

### 修正公式
```
if (spec_type == 0 && spec_param == HSS_SORCERY && sorcery_skill > 0):
    effective_pct = sorcery_base_pct × (1 + hero_level / 20.0)
else:
    effective_pct = sorcery_base_pct

damage = (int)((double)damage × (1.0 + effective_pct))
```

### 验证
- 58 级、专家魔力（sorcery=3 → base=0.15）：
  - `0.15 × (1 + 58/20) = 0.15 × 3.9 = 0.585`
  - `1890 × 1.585 = 2995.65 → 2995` ✓
- 无 Sorcery 技能时：sorcery_base=0，特长不生效 ✓
- 魔力特长必须要有魔力辅助技能才生效 ✓

### homm3.h 修正说明
- `hero->power` 定义在 +1142(0x476)，和 `attack` 同偏移——这是 homm3.h 的定义错误
- 实际主属性在 0x476~0x479（ATK/DEF/SPW/KNO），spell power 在 0x478
- 当前 hero 的 ATK 和 SPW 巧合都是不同值(69/92)，但 `hero->power` 读到的 92 恰好是正确的 SPW 值
- 代码改为直接从 `raw[0x478]` 读 spell power 以确保正确
