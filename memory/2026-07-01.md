## H3BattleValueInfo 背景图渲染问题解决

### 问题
HD_TC2 hook 了 `_Pcx8_::DrawToPcx16`（0x44FA80），自定义 `_Pcx8_` 对象调用该函数会崩溃（EXCEPTION_ACCESS_VIOLATION, read 0x3BC）。
直接写 `screen_pcx16->buffer` 也不行——HD_TC2 渲染管线会覆盖，导致颜色和位置错误。

### 尝试过的失败方案
1. 自定义 `_Pcx8_` + `DrawToPcx16` → HD_TC2 崩溃
2. 手写 `BlitPcx8ToPcx16` 直接写 screen buffer → 颜色/位置错乱
3. 注入 `o_BinTreeHead` 资源树 → 仍然崩溃
4. 转为 `_Pcx16_` 中间 surface → 同样写 buffer 问题

### 最终方案：DirectDraw Blt
- PCX 解码 → 创建 DD 离屏 surface → Lock 写像素（自动适配 16/32-bit）→ `o_DDSurfaceBackBuffer->Blt()`
- 需要 `#include <ddraw.h>` 在 `#include "homm3.h"` 之前
- 成功显示自定义背景图，颜色位置正确
