## 任务一：确认信息窗口的文本追加方式

**目标**：找到追加文本行的函数地址、对话框高度字段偏移，以及 Hook 的精确地址。

**已通过 radare2 静态分析确认的结构（`sub_468440`）：**

```
0x4684f0  call 0x5f3700   ; 构建对话框内容，返回对话框对象指针→ ESI
0x468505  mov ecx, esi
0x468507  je   0x468510
0x468509  call 0x5f4b90   ; 显示窗口（有部队时走此路径）←─ Hook插入点在这之前
0x46850e  jmp  0x468546
0x468510  call 0x5f4ba0   ; 显示窗口（无部队时）
```

**Hook 精确地址：`0x468509`**（`call 0x5f4b90` 处，此时 `ECX/ESI` = 对话框对象指针）

**文本控件构造函数：`0x5bc6a0`**

调用参数（从 `sub_5f3700` 内的调用处逆推，__thiscall，ECX=新控件对象）：

```cpp
// push 顺序（右→左）：
sub_5bc6a0(
  ecx,          // 新控件对象（this）
  x,            // 0x14 = 20
  y,            // 0xe8 = 232（或其他行的y值）
  width,        // 0xc0 = 192
  height,       // 0x29 = 41
  text_ptr,     // 字符串指针
  font_name,    // "smalfont.fnt"
  align,        // 4
  color,        // 0xffffffff
  unk1,         // 0
  unk2,         // 0
  unk3          // 8
)
```

**AddItem 函数：`0x5fe2d0`**（__thiscall，ECX=对话框对象）

```cpp
sub_5fe2d0(ecx=dlg, item_ptr, 1);
```

**对话框基类构造（`0x5ffc00`）写入的字段偏移：**

| 偏移 | 含义 | 初始值 |
|------|------|--------|
| `+0x10` | height（像素） | 由参数决定 |
| `+0x18` | x |  |
| `+0x1c` | y |  |
| `+0x20` | width |  |
| `+0x24` | height（同上另一处）|  |

> `+0x10` 是追加行后需要增大的高度字段，每追加一行增加约 `行高+间距` 像素。

### 还需 x32dbg 动态确认

1. 在 `0x468509` 下断点，触发信息窗口
2. 此时 `ESI` = 对话框对象指针，在内存窗口查看 `[ESI+0x10]` 的值（当前高度）
3. 记录该值，确认 `+0x10` 确实是高度字段

### 产出汇总

```
Hook 地址：          0x468509（call 0x5f4b90 前，ESI=对话框指针）
文本控件构造：       0x5bc6a0(ecx=item, x, y, w, h, text, font, align, color, ...)
AddItem：           0x5fe2d0(ecx=dlg, item, 1)
对话框高度字段偏移： +0x10（待 x32dbg 动态确认）
```

## 任务二：定位战场空白区域右键分发入口

**已通过 radare2 静态分析完成，无需动态调试。**

### 分析结果

**战场消息分发函数：`0x4746b0`**（__thiscall，ECX=BattleMgr*，第一参数=msg*）

消息结构体字段偏移（`eax = msg*`）：

| 偏移 | 字段 | 说明 |
|------|------|------|
| `+0x0` | 主类型 | dec后=3→鼠标移动，0x200→其他 |
| `+0x4` | subtype | **0xE = MST_RBUTTONDOWN**（右键）|
| `+0x8` | BattleStack* | 0=空格子，非0=有部队 |
| `+0x10` | x_abs | 鼠标x坐标 |
| `+0x14` | y_abs | 鼠标y坐标 |

**关键代码路径（右键分支）：**

```
0x4746f3  cmp ecx, 0xc
0x4746f6  je  0x474701
0x4746f8  cmp ecx, 0xe      ← subtype == 右键
0x4746fb  jne 0x475764      ← 不是右键则退出
0x474701  mov ecx, [eax+8]  ← BattleStack*
0x474706  je  0x474727      ← 空格子路径（BattleStack==0）
0x47470b  ...               ← 有部队路径 → 最终call 0x468440
0x474727  call 0x4762f0     ← 空格子：走HD Mod消息分发
```

**空格子插入点：`0x474727`**（`je 0x474727` 命中时，即右键+空格子），此处 Hook 可消费消息并弹出自定义窗口。

**GetCreatureAtHex：`0x4e7230`**（从 sub_4762f0 中找到，参数为 BattleStack 结构内的 hex 坐标）

### 产出汇总

```
战场消息分发函数地址：  0x4746b0
消息 subtype 偏移：     +0x4（0xE = 右键）
消息 x_abs 偏移：       +0x10
消息 y_abs 偏移：       +0x14
BattleStack* 偏移：     +0x8（0=空格子）
空格子 Hook 地址：      0x474727（右键+空格子入口）
Hook 方式：             WriteLoHook(0x474727) 拦截，弹出远程力量窗口后 EXEC_DEFAULT 或 NO_EXEC_DEFAULT
```
