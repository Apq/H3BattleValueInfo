# 远程输出能力算法

> 日期：2026-07-01  
> 实现文件：`modules/CreatureDialog.inc.cpp`  
> 相关函数：`CalcRangedUnitsPower()`、`CalcRangedOutputToTarget()`、`CalcHeroSpellPower()`、`GetHeroEffectiveSpellLevel()`

## 1. 面板含义

战场顶部面板当前显示三行：

```text
远程输出
魔法输出
总输出
```

两列分别表示战斗双方。

当前定义：

```text
远程输出 = 当前一方远程单位在所有敌方存活目标上的保守射击输出
魔法输出 = 当前一方英雄已掌握白名单攻击魔法中的最高理论单次伤害
总输出   = 远程输出 + 魔法输出
```

## 2. 远程输出定义

远程输出表示一方远程单位在当前战场状态下的保守输出能力：

```text
远程输出能力 = min(本方全部远程单位对每一支敌方存活部队的总射击输出)
```

即：逐一把敌方每支存活部队作为目标，计算本方所有可射击远程部队对该目标的合计伤害，最后取最小值。

这不是最佳目标输出，也不是 AI 实际目标选择，而是当前战场状态下的保守输出下限。

## 3. 远程输出总流程

当前实现：

```cpp
static int CalcRangedUnitsPower(_BattleMgr_* mgr, int side)
{
    if (!mgr || side < 0 || side > 1) return 0;

    int enemy_side = 1 - side;
    bool has_target = false;
    int min_output = 0x7FFFFFFF;

    for (int slot = 0; slot < 21; ++slot) {
        _BattleStack_* target = &mgr->stack[enemy_side][slot];
        if (target->count_current <= 0) continue;

        has_target = true;
        int output = CalcRangedOutputToTarget(mgr, side, target);
        if (output < min_output) min_output = output;
    }
    return has_target ? min_output : 0;
}
```

规则：

1. `side` 只能是 `0` 或 `1`。
2. 遍历敌方 21 个 stack 槽位。
3. 只把 `count_current > 0` 的敌方 stack 作为目标。
4. 对每个目标调用 `CalcRangedOutputToTarget()`。
5. 返回所有目标结果中的最小值。
6. 如果敌方没有存活目标，返回 `0`。

## 4. 单目标远程输出

当前实现：

```cpp
static int CalcRangedOutputToTarget(_BattleMgr_* mgr, int side, _BattleStack_* target)
{
    if (!mgr || side < 0 || side > 1 || !target || target->count_current <= 0) return 0;

    __int64 output = 0;
    for (int slot = 0; slot < 21; ++slot) {
        _BattleStack_* shooter = &mgr->stack[side][slot];
        if (shooter->count_current <= 0 || shooter->creature.shots <= 0) continue;

        if (!shooter->CanShoot(target)) continue;

        int base_damage = CALL_2(int, __thiscall, 0x442E80, shooter, 0);
        if (base_damage <= 0) continue;

        int fireshield_damage = 0;
        int damage = shooter->Calc_Damage_Bonuses(target, base_damage, TRUE, TRUE, 0, &fireshield_damage);
        if (damage > 0) output += damage;
    }
    if (output > 0x7FFFFFFF) return 0x7FFFFFFF;
    return (int)output;
}
```

规则：

1. 遍历本方 21 个 stack 槽位。
2. 跳过死亡 stack：`count_current <= 0`。
3. 跳过无射击次数的 stack：`creature.shots <= 0`。
4. 调用原版 `CanShoot(target)` 判断是否能射击目标。
5. 不能射击时，该 shooter 对该目标贡献 `0`。
6. 能射击时调用原版 `0x442E80` 计算基础伤害。
7. 再调用原版 `Calc_Damage_Bonuses()` 计算修正后射击伤害。
8. 正伤害累加到 `output`。
9. 内部累加使用 `__int64`，最终超过 `0x7FFFFFFF` 时钳制到 `0x7FFFFFFF`。

## 5. 远程输出依赖的原版函数

### `CanShoot(target)` / `0x442610`

用于判断当前远程 stack 是否能射击指定目标。

交给原版处理的规则包括：

1. 贴脸限制。
2. 障碍物。
3. 城墙。
4. 弹药。
5. 幻影神弓、金弓等射击可行性相关效果。

### `_BattleStack_::CalcBaseDamage(0)` / `0x442E80`

用于计算基础伤害。

当前代码传参：

```cpp
CALL_2(int, __thiscall, 0x442E80, shooter, 0)
```

交给原版处理的规则包括：

1. 当前数量。
2. 伤害上下限。
3. 祝福。
4. 诅咒。
5. 其他影响基础伤害的状态。

### `_BattleStack_::Calc_Damage_Bonuses(...)` / `0x443C60`

当前调用：

```cpp
shooter->Calc_Damage_Bonuses(target, base_damage, TRUE, TRUE, 0, &fireshield_damage);
```

参数含义：

```text
target            = 目标 stack
base_damage       = 原版基础伤害
IsShot = TRUE     = 按射击攻击计算
Virtual = TRUE    = 虚拟计算，不实际造成伤害
WayLength = 0     = 射击不使用骑兵移动距离加成
fireshield_damage = 接收火盾伤害，但当前远程输出不使用它
```

交给原版处理的规则包括：

1. 攻防差。
2. 英雄攻防。
3. 状态后的攻防。
4. 箭术。
5. 距离惩罚。
6. 城墙惩罚。
7. 宝物和状态造成的伤害修正。

## 6. 魔法输出定义

魔法输出不按敌方目标逐一计算，只看当前英雄自身能提供的攻击魔法上限。

前提：

1. 本方必须有英雄。
2. 英雄必须装备魔法书。
3. 不检查当前魔法值。
4. 不检查本回合是否已经施法。
5. 不考虑敌方免疫、抗魔、亡灵/非亡灵、站位、范围命中数量等目标相关因素。

当前定义：

```text
魔法输出能力 = 当前英雄已掌握的、计入白名单的攻击魔法中，理论伤害最高的那个值
```

## 8. 有效法术等级

当前代码不再使用：

```cpp
hero->spell_level[spell_id]
```

现在使用原版函数 `0x4E52F0` 获取有效法术等级：

```cpp
static int GetHeroEffectiveSpellLevel(_Hero_* hero, int spell_id)
{
    if (!hero || spell_id < 0 || spell_id >= 70) return 0;

    int land_modifier = hero->GetLandModifierUnder();
    int level = CALL_3(int, __thiscall, 0x4E52F0, hero, spell_id, land_modifier);
    if (level < 0) level = 0;
    if (level > 3) level = 3;
    return level;
}
```

`0x4E52F0` 的调用约定已确认：

```text
this = hero
arg1 = spell_id
arg2 = land_modifier
return = effective spell level, 0..3
```

这个原版函数会综合：

1. 英雄气系魔法等级。
2. 英雄水系魔法等级。
3. 英雄火系魔法等级。
4. 英雄土系魔法等级。
5. 魔法地形修正。
6. 原版对部分法术的特殊规则。

## 9. 魔法伤害公式

当前实现：

```cpp
int level = GetHeroEffectiveSpellLevel(hero, spell_id);

_Spell_& spell = o_Spell[spell_id];
int damage = spell.effect[level] + spell.eff_power * spell_power;

damage += hero->GetSpell_Specialisation_Bonuses(spell_id, 1, damage);

int sorcery = hero->second_skill[HSS_SORCERY];
if (sorcery == 1) damage = damage * 105 / 100;
else if (sorcery == 2) damage = damage * 110 / 100;
else if (sorcery >= 3) damage = damage * 115 / 100;

damage = ApplySpellDamageArtifactBonuses(hero, spell_id, damage);
```

其中：

```cpp
int spell_power = hero->power;
if (spell_power < 1) spell_power = 1;
```

也就是：

```text
基础魔法伤害 = o_Spell[spell_id].effect[有效法术等级]
             + o_Spell[spell_id].eff_power * max(hero->power, 1)
```

然后依次追加：

1. 英雄法术特长加成。
2. 魔力 Sorcery 二级技能加成。
3. 宝物造成的最终法术伤害修正。

## 10. 英雄法术特长

当前调用原版函数：

```cpp
hero->GetSpell_Specialisation_Bonuses(spell_id, 1, damage)
```

对应地址：

```text
0x4E6260
```

原版会按英雄等级、英雄特长类型、特长对象计算额外加成。

由于当前魔法输出定义为“不考虑具体敌方目标”，没有目标生物等级可传，所以实现中无目标估算参数固定用 `1`。

## 12. 宝物法术伤害修正

当前已计入四系灵球。实现函数：

```cpp
static int ApplySpellDamageArtifactBonuses(_Hero_* hero, int spell_id, int damage)
{
    if (!hero || spell_id < 0 || spell_id >= 70 || damage <= 0) return damage;

    const unsigned school = o_Spell[spell_id].school_flags;
    bool boosted = false;
    if ((school & SSF_AIR) && hero->DoesWearArtifact(AID_ORB_OF_THE_FIRMAMENT)) boosted = true;
    if ((school & SSF_EARTH) && hero->DoesWearArtifact(AID_ORB_OF_SILT)) boosted = true;
    if ((school & SSF_FIRE) && hero->DoesWearArtifact(AID_ORB_OF_TEMPESTUOUS_FIRE)) boosted = true;
    if ((school & SSF_WATER) && hero->DoesWearArtifact(AID_ORB_OF_DRIVING_RAIN)) boosted = true;

    if (boosted) damage = damage * 150 / 100;
    return damage;
}
```

对应关系：

| 宝物常量 | ID | 学派 | 效果 |
|---|---:|---|---|
| `AID_ORB_OF_THE_FIRMAMENT` | 79 | 气 `SSF_AIR` | 对应学派法术伤害 +50% |
| `AID_ORB_OF_SILT` | 80 | 土 `SSF_EARTH` | 对应学派法术伤害 +50% |
| `AID_ORB_OF_TEMPESTUOUS_FIRE` | 81 | 火 `SSF_FIRE` | 对应学派法术伤害 +50% |
| `AID_ORB_OF_DRIVING_RAIN` | 82 | 水 `SSF_WATER` | 对应学派法术伤害 +50% |

判断依据是 `o_Spell[spell_id].school_flags`。如果一个法术命中多个学派且英雄装备多个对应灵球，当前实现也只加成一次，避免重复叠乘。


最终显示：

```text
总输出 = 远程输出 + 魔法输出
```

## 14. Overlay Panel 与刷新策略

当前对比框实现为独立 overlay panel：`RangedOverlayPanel`。

它不是 HoMM3 的独立 `_Dlg_` 窗口，也不向战斗对话框插入 `_DlgItem_`。它有独立的生命周期、背景资源、文本缓存和 dirty 状态，但绘制时只画到屏幕层，不抢输入、不改变 `o_CurrentDlg`。

位置保持旧逻辑：水平居中于战斗窗口，吸附在战斗窗口外侧上方。

当前面板使用事件驱动的脏标记，不再每次绘制都完整重算。

置脏时机：

1. 进入战斗后：`Combat_StartBattle @ 0x4781C0`。
2. 任一方施法后：`MagicSystem_2 @ 0x464F10`。
3. 任意 stack 行动处理后：`FUN_004746B0`。

实现方式：

```cpp
class RangedOverlayPanel
{
public:
    void MarkDirty(const char* reason);
    void Destroy();
    void Draw(_BattleMgr_* mgr);

private:
    void Recalculate(_BattleMgr_* mgr, const char* reason);

    _Pcx8_* bg_;
    _Dlg_* last_dlg_;
    _BattleMgr_* last_mgr_;
    bool active_;
    bool dirty_;
    unsigned int stack_checksum_;
    char text_[6][64];
};
```

外部只通过三个入口操作它：

```cpp
static void MarkRangedPanelDirty(const char* reason)
{
    s_ranged_overlay_panel.MarkDirty(reason);
}

static void DestroyRangedPanel()
{
    s_ranged_overlay_panel.Destroy();
}

static void UpdateRangedPanel(_BattleMgr_* mgr)
{
    s_ranged_overlay_panel.Draw(mgr);
}
```

绘制时只消费 overlay panel 自己的缓存：

```cpp
if (dirty_ || !text_[0][0]) {
    Recalculate(mgr, dirty_ ? "dirty" : "empty cache");
}
...绘制 text_...
```

也就是说：

1. 正常帧只绘制缓存文本。
2. 进入战斗后首次绘制会计算一次。
3. 部队行动后下一次绘制会计算一次。
4. 任一方施法后下一次绘制会计算一次。
5. 不再依赖 stack checksum 判断是否刷新。

`ComputeStackChecksum(mgr)` 仍保留并写入日志，主要用于诊断，不再作为刷新条件。

## 15. 当前算法特点

1. 远程输出尽量使用原版函数，不手写射击规则。
2. 远程输出随战场状态实时变化。
3. 远程输出对敌方每支存活部队逐一试算。
4. 远程输出取最小值，表示保守输出下限。
5. 魔法输出只估算英雄攻击魔法上限，不绑定目标。
6. 魔法输出已使用原版有效法术等级 `0x4E52F0`。
7. 魔法输出已计入英雄法术特长和魔力 Sorcery。
8. 魔法输出已计入四系灵球造成的对应学派最终伤害 +50%。
9. 魔法输出已计入诅咒之地（双方限 1 级）和禁魔披风（佩戴方限 1-2 级）的限制。
10. 面板使用事件驱动脏标记：进入战斗、stack 行动后、任一方施法后才重算。
11. 正常绘制帧只画缓存文本，避免每帧完整计算。
12. 总输出只是远程输出和魔法输出的直接相加，不代表实际单回合必然可打出的总伤害。

## 16. 当前未计入项

远程输出部分主要依赖原版战斗伤害函数，未主动额外展开解释所有原版内部规则。

魔法输出当前未计入：

1. 当前魔法值是否足够。
2. 本回合是否已经施法。
3. 目标魔法抗性。
4. 目标魔法免疫。
5. 目标阵营/种族限制。
6. 范围法术实际命中数量。
7. 战场站位对范围法术收益的影响。
8. 四系灵球之外的其他宝物、地形或特殊插件可能额外修改的最终法术伤害。

**已计入**（见 §17-18）：禁魔地形（诅咒之地）、禁魔披风。

因此魔法输出是“英雄可用攻击魔法的理论单体上限估算”，不是完整战斗施法模拟。

## 17. 禁魔地形

战场地形类型存储在 `_BattleMgr_->spec_terr_type`。

| `spec_terr_type` | 地形 | 对魔法输出的影响 |
|---:|---|---|
| `-1` | 无特殊地形 | 正常 |
| `0` | 海岸（Coast） | 正常 |
| `1` | 魔法平原（Magic Plains） | 所有法术视为专家级（level=3），不需魔法技能 |
| `2` | 诅咒之地（Cursed Ground） | **双方只能施放 1 级魔法**，2 级及以上被禁止 |

### 实现

在 `CalcHeroSpellPower()` 中：

```cpp
bool cursed_ground = (mgr->spec_terr_type == 2);
int max_spell_level = 5;
if (cursed_ground) max_spell_level = 1;
```

遍历白名单法术时跳过超等级法术：

```cpp
if (spell.level > max_spell_level) continue;
```

魔法平原的影响由原版 `0x4E52F0` (`GetEffectiveSpellLevel`) 处理，
因为该函数接收 `land_modifier` 参数。

## 18. 禁魔披风（Recanter's Cloak）

宝物 ID：`AID_RECANTERS_CLOAK = 83`

效果：**佩戴该宝物的英雄在战斗中只能施放 1-2 级魔法**。

### 关键规则

- 只限制**佩戴方自己的英雄**，不影响对方。
- 3 级及以上魔法被完全禁止。
- 1 级和 2 级魔法不受影响。

### 与诅咒之地对比

| 限制源 | 最高可用法术等级 |
|---|---:|
| 无限制 | 5 |
| 禁魔披风（仅佩戴方） | 2 |
| 诅咒之地（双方） | 1 |

### 实现

在 `CalcHeroSpellPower()` 中，与诅咒之地取更严限制：

```cpp
bool recanter = hero->DoesWearArtifact(AID_RECANTERS_CLOAK);
int max_spell_level = 5;
if (cursed_ground) max_spell_level = 1;
else if (recanter) max_spell_level = 2;
```
