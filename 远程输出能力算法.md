# 远程/魔法输出能力算法

> 实现文件：`modules/CreatureDialog.inc.cpp`、`modules/Compat.inc.cpp`

战斗顶部面板显示双方三项数值：

```text
远程输出
魔法输出
总输出
```

两列分别对应战斗双方。

## 定义

远程输出：

```text
min(本方所有可射击远程 stack 对敌方每一支存活 stack 的合计射击伤害)
```

魔法输出：

```text
本方英雄已掌握白名单攻击魔法中的最高理论单次伤害
```

总输出：

```text
远程输出 + 魔法输出
```

这三个值都是“输出能力”估算，不代表本回合一定能实际打出的总伤害。

## 远程输出

远程输出是保守值。对敌方每支存活 stack 逐一作为目标试算，本方所有可射击远程 stack 对该目标的伤害求和，然后取这些目标结果中的最小值。

### 一方远程输出

当前代码结构：

```cpp
static int CalcRangedUnitsPower(_BattleMgr_* mgr, int side)
{
    int enemy_side = 1 - side;
    bool has_target = false;
    int min_output = 0x7FFFFFFF;

    for (int slot = 0; slot < 21; ++slot) {
        _BattleStack_* target = &mgr->stack[enemy_side][slot];
        if (target->count_current <= 0) continue;

        has_target = true;
        int output = CalcRangedOutputToTarget(mgr, side, target);
        if (output < min_output) min_output = output;
    }

    return has_target ? min_output : 0;
}
```

规则：

1. `side` 只能是 `0` 或 `1`。
2. 遍历敌方 21 个 stack 槽位。
3. 只把 `count_current > 0` 的敌方 stack 作为目标。
4. 对每个目标调用 `CalcRangedOutputToTarget()`。
5. 返回所有目标结果中的最小值。
6. 如果敌方没有存活目标，返回 `0`。

### 单目标输出

当前代码结构：

```cpp
static int CalcRangedOutputToTarget(_BattleMgr_* mgr, int side, _BattleStack_* target)
{
    __int64 output = 0;

    for (int slot = 0; slot < 21; ++slot) {
        _BattleStack_* shooter = &mgr->stack[side][slot];
        if (shooter->count_current <= 0 || shooter->creature.shots <= 0) continue;
        if (shooter->blinded || shooter->paralyzed || shooter->forgetfulness_level > 0) continue;

        bool can_shoot = SafeCanShoot(shooter, target);
        int base_damage = can_shoot ? SafeCalcBaseDamage(shooter) : 0;
        if (!can_shoot || base_damage <= 0) continue;

        int fireshield_damage = 0;
        int damage = SafeCalcDamageBonuses(shooter, target, base_damage, &fireshield_damage);
        if (damage > 0) output += damage;
    }

    if (output > 0x7FFFFFFF) return 0x7FFFFFFF;
    return (int)output;
}
```

规则：

1. 遍历本方 21 个 stack 槽位。
2. 跳过死亡 stack：`count_current <= 0`。
3. 跳过无射击次数 stack：`creature.shots <= 0`。
4. 跳过当前失能或无法远程攻击的射手：`blinded`、`paralyzed`、`forgetfulness_level > 0`。已经行动过的远程 stack 仍计入输出能力。
5. 调用原版 `CanShoot(target)` 判断当前战场状态下能否射击目标。
6. 调用原版 `CalcBaseDamage(0)` 计算基础伤害。
7. 调用原版 `Calc_Damage_Bonuses(..., IsShot=TRUE, Virtual=TRUE)` 计算修正后射击伤害。
8. 正伤害累加。
9. 内部使用 `__int64` 累加，最终钳制到 `0x7FFFFFFF`。

### 远程依赖的原版函数

| 地址 | 包装函数 | 用途 |
|---:|---|---|
| `0x442610` | `SafeCanShoot` | 判断 shooter 是否能射击 target |
| `0x442E80` | `SafeCalcBaseDamage` | 计算基础伤害 |
| `0x443C60` | `SafeCalcDamageBonuses` | 计算攻防、箭术、距离、城墙、宝物等修正后的伤害 |

这些调用都在独立 Safe 包装函数中使用 SEH 保护。面板是附加 UI，任何原版数据未就绪或第三方 hook 异常都不应带崩游戏。

## 魔法输出

魔法输出只看英雄自身能提供的攻击魔法上限，不绑定具体敌方目标。

前提：

1. 本方有英雄。
2. 英雄装备魔法书：`bodyArtifacts[17]`。
3. 英雄已掌握对应白名单魔法。
4. 不检查当前魔法值。
5. 不检查本回合是否已经施法。
6. 不考虑敌方免疫、抗魔、亡灵/非亡灵、站位、范围命中数量等目标因素。

### 白名单

| 常量 | ID | 等级 | 英文名 | 常见中文名 |
|---|---:|---:|---|---|
| `SPL_MAGIC_ARROW` | 15 | 1 | Magic Arrow | 魔法神箭 |
| `SPL_ICE_BOLT` | 16 | 2 | Ice Bolt | 霹雳寒冰 |
| `SPL_LIGHTNING_BOLT` | 17 | 2 | Lightning Bolt | 霹雳闪电 |
| `SPL_IMPLOSION` | 18 | 5 | Implosion | 雷鸣爆弹 |
| `SPL_FROST_RING` | 20 | 3 | Frost Ring | 寒冰魔环 |
| `SPL_FIREBALL` | 21 | 3 | Fireball | 连珠火球 |
| `SPL_METEOR_SHOWER` | 23 | 4 | Meteor Shower | 流星火雨 |
| `SPL_TITANS_LIGHTNING_BOLT` | 57 | 5 | Titan's Lightning Bolt | 泰坦之箭 |

当前不计入目标强依赖或规则特殊的法术，例如地雷、火墙、末日审判、死亡波纹、毁灭亡灵。

### 有效法术等级

当前不直接使用 `hero->spell_level[spell_id]` 作为最终等级，而是调用原版函数：

```cpp
SafeGetEffectiveSpellLevel(hero, spell_id, land_modifier)
```

相关地址：

| 地址 | 用途 |
|---:|---|
| `0x4E5210` | 获取英雄脚下地形魔法修正 |
| `0x4E52F0` | 获取有效法术等级 |

原版函数会综合四系魔法技能、魔法地形和部分法术特殊规则。返回值限制到 `0..3`。

### 伤害公式

核心公式：

```text
damage = spell.effect[level] + spell.eff_power * max(hero->power, 1)
```

然后依次处理：

1. 英雄法术特长加成：`0x4E6260`。
2. 魔力 Sorcery 副技能加成。
3. 四系灵球最终伤害加成。

### 魔力 Sorcery

`HSS_SORCERY = 25`。

基础加成：

| 等级 | 加成 |
|---:|---:|
| 1 | +5% |
| 2 | +10% |
| 3 | +15% |

如果英雄特长是魔力，当前实现会按英雄等级增强 Sorcery 效果：

```text
effective_pct = base_pct * (1 + hero_level / 20.0)
```

无 Sorcery 技能时，魔力特长不单独生效。

### 宝物修正

当前计入：

| 宝物常量 | ID | 学派 | 效果 |
|---|---:|---|---|
| `AID_ORB_OF_THE_FIRMAMENT` | 125 | 气 | 对应学派法术伤害 +50% |
| `AID_ORB_OF_SILT` | 126 | 土 | 对应学派法术伤害 +50% |
| `AID_ORB_OF_TEMPESTUOUS_FIRE` | 127 | 火 | 对应学派法术伤害 +50% |
| `AID_ORB_OF_DRIVING_RAIN` | 128 | 水 | 对应学派法术伤害 +50% |

判断依据是 `o_Spell[spell_id].school_flags`。一个法术即使命中多个学派且英雄装备多个对应灵球，当前实现也只加成一次，避免重复叠乘。

另外：

- 禁魔披风 `AID_RECANTERS_CLOAK = 83`：佩戴方只能施放 1-2 级魔法。
- 诅咒之地：`spec_terr_type == 2`，双方只能施放 1 级魔法。
- 魔法平原：`spec_terr_type == 1`，有效等级由原版 `GetEffectiveSpellLevel` 处理。

## 结构要求

这些结构大小和偏移必须保持正确，否则会出现全 0、垃圾值或崩溃。

| 结构/字段 | 当前要求 |
|---|---|
| `_BattleStack_` | 大小必须为 `0x548` |
| `_BattleStack_.creature` | `0x74` |
| `_CreatureInfoCompat.hit_points` | `0x4C` |
| `_CreatureInfoCompat.shots` | `0x64` |
| `_Hero_.bodyArtifacts[19]` | `0x12D`，每项 8 字节 |
| Spell book 槽位 | `bodyArtifacts[17]` |
| `_Hero_.spell[70]` | `0x3EA` |
| `_Hero_.power` | `0x478` |
| `_Spell_` | 大小必须为 `0x88`，否则 `o_Spell[spell_id]` 步进错误 |
| `_BattleMgr_.spec_terr_type` | `0x53C0` |
| `_BattleMgr_.stack[2][21]` | `0x54CC` |

## 刷新策略

面板维护文本缓存和 stack checksum。

重算时机：

1. 首次显示。
2. `dirty_` 被置位。
3. 当前 stack checksum 与上次不同。

置脏入口：

| 事件 | Hook |
|---|---|
| 进入手动战斗 | `Combat_StartBattle @ 0x4781C0` |
| 施法后 | `MagicSystem_2 @ 0x464F10` |

当前没有注册 `0x4746B0` 行动 hook；栈变化由 checksum 自动发现。

checksum 纳入：

- `spec_terr_type`
- 双方英雄指针、魔法书、法强、知识、魔力技能、已学魔法
- 双方是否已施法、当前行动 stack、当前行动方、移动/攻击相关字段
- stack 数量、位置、损血、弹药、攻防、伤害、HP 等基础字段
- bless/curse/precision/forgetfulness/blind/paralyze 等关键法术状态
- bloodlust、weakness、prayer、slayer、frenzy、shield、air shield 等影响伤害的状态
- `is_done`、`blinded`、`paralyzed`、`forgetfulness_level` 等状态；`is_done` 仅用于触发刷新，不用于排除远程 stack

## 局限

远程输出依赖原版函数处理细节，但仍是虚拟估算：

1. 不代表 AI 或玩家实际会选择哪个目标。
2. 对每个敌方目标取最小值，刻意偏保守。
3. 只按当前战场状态估算，不预测后续行动；已经行动的远程 stack 仍计入输出能力，失明、麻痹、遗忘等无法射击状态不计入。

魔法输出未计入：

1. 当前魔法值是否足够。
2. 本回合是否已经施法。
3. 目标免疫、抗魔、抗性宝物。
4. 范围法术命中多个目标的总收益。
5. 敌我阵营限制，例如亡灵、非亡灵等特殊目标规则。
