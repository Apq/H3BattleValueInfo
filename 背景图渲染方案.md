# 战斗面板背景图渲染方案

战斗顶部输出面板使用 DirectDraw 离屏 surface 渲染，绕开 HoMM3 原始 `_Pcx8_/_Pcx16_` 绘制链路。

## 背景

HD_TC2（HD Mod 真彩模块）hook 了游戏原始 `_Pcx8_::DrawToPcx16`（`0x44FA80`）。自定义创建的 `_Pcx8_` 对象即使注册到资源树，也会在 HD_TC2 hook 内访问无效内部状态并崩溃。

直接写 `screen_pcx16->buffer` 也不可行。HD_TC2 改变了屏幕渲染管线，游戏逻辑层 `_Pcx16_` buffer 不再等同最终显示的 backbuffer，手写像素会出现颜色、位置或显示时序问题。

## 当前方案

当前完全绕开 `_Pcx8_/_Pcx16_` 图像绘制，直接使用 DirectDraw：

```text
PCX 文件
  -> 手动解码 1-plane 8-bit PCX + 调色板
  -> o_DD->CreateSurface() 创建离屏 surface
  -> Lock surface 写入 RGB565 或 32-bit 像素
  -> o_DDSurfaceBackBuffer->Blt() 画到 backbuffer
```

相关全局变量：

| 全局变量 | 地址 | 类型 |
|---|---:|---|
| `o_DD` | `0x6AAD20` | `LPDIRECTDRAW` |
| `o_DDSurfacePrimary` | `0x6AAD24` | 主 surface |
| `o_DDSurfaceBackBuffer` | `0x6AAD28` | backbuffer |

## PCX 支持范围

当前只支持：

- 1-plane
- 8-bit indexed color
- 文件尾含 768 字节调色板

配置项：

```ini
[RangedPanel]
BackgroundImage=rp_bg.pcx
```

图片放在插件目录的 `img` 子目录。

## 面板绘制流程

1. `CreateDDBackground()` 读取并解码 PCX。
2. 创建离屏 DirectDraw surface。
3. 根据实际 surface 像素格式写入：
   - 16-bit：RGB565
   - 32-bit：ARGB/BGRA 风格 32-bit 像素
4. `DrawDDBackground()` 使用 `Blt` 把背景贴到 backbuffer。
5. 文本仍使用原版字体 `TextDraw` 绘制。

## 面板隐藏与残留处理

面板位于 800x600 战场框外部上方。快速战斗结果界面或战斗结果界面不会可靠覆盖这一区域，因此隐藏时不能靠结果界面或战场重绘自然清掉。

也不能使用黑色填充，黑块会留在结果界面上。

当前实现使用 underlay 备份：

1. 每次画面板前，先把面板所在矩形下方的 backbuffer 原始像素复制到离屏 surface。
2. 只保存一次，避免后续帧把面板自身保存进去。
3. suppress/destroy 时，把保存的原始像素 Blt 回 backbuffer。
4. 释放 underlay surface。

相关函数：

| 函数 | 用途 |
|---|---|
| `CreateEmptyDDSurface()` | 创建空离屏 surface |
| `CopyBackBufferToSurface()` | 从 backbuffer 保存面板底下原始像素 |
| `RestoreSurfaceToBackBuffer()` | 把保存的原始像素恢复到 backbuffer |
| `SaveUnderlay()` | 面板绘制前保存 underlay |
| `RestoreUnderlay()` | 面板隐藏/销毁时恢复 underlay |

## 不工作的方案

| 方案 | 结果 | 原因 |
|---|---|---|
| 自定义 `_Pcx8_::DrawToPcx16` | 崩溃 | HD_TC2 hook 访问自定义 Pcx8 缺失的内部状态 |
| 注册到 `o_BinTreeHead` 资源树 | 崩溃 | HD_TC2 检查的不只是资源树归属 |
| 手写 `screen_pcx16->buffer` | 显示错乱 | HD_TC2 使用自己的最终渲染管线 |
| `_Pcx16_::DrawPcx16` 风格手动写 buffer | 显示错乱 | 本质仍是写逻辑 buffer |
| `DDBLT_COLORFILL` 填黑清除 | 出现黑块 | 面板区域在结果界面覆盖范围外 |
| suppress 后调用 `RedrawBattlefield` | 不可靠 | 面板在战场框外部，战场重绘不保证覆盖该区域 |

## 注意事项

- `#include <ddraw.h>` 必须在 `homm3.h/H3API.hpp` 相关头之前可用。
- DirectDraw 调用都应容错，失败只影响面板显示，不应带崩游戏。
- underlay 必须在面板首次绘制前保存，不能每帧重复覆盖。
- 坐标或尺寸变化时，应先恢复旧 underlay，再保存新 underlay。
