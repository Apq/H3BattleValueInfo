# 战斗面板背景图渲染方案

战斗顶部输出面板使用 DirectDraw 离屏 surface 渲染背景，文字使用原版 `TextDraw` 画到 `screen_pcx16`。两者是不同的输出目标，需要同步处理。

## 背景

HD_TC2（HD Mod 真彩模块）hook 了游戏原始 `_Pcx8_::DrawToPcx16`（`0x44FA80`）。自定义创建的 `_Pcx8_` 对象即使注册到资源树，也会在 HD_TC2 hook 内访问无效内部状态并崩溃。

直接写 `screen_pcx16->buffer` 作为背景也不可行。HD_TC2 改变了屏幕渲染管线，手写背景像素会出现颜色、位置或显示时序问题。

## 双缓冲架构

面板绘制涉及两个独立输出目标：

| 输出目标 | 内容 | 绘制方式 |
|---|---|---|
| DD backbuffer | 面板背景图 | `o_DDSurfaceBackBuffer->Blt()` |
| `screen_pcx16` | 面板文字 | `_Fnt_::TextDraw()` |

HD Mod 渲染流程：

```text
游戏画战场 -> screen_pcx16
HD_TC2 翻转 -> screen_pcx16 到 DD backbuffer
DD backbuffer 翻转 -> 屏幕显示
```

面板 hook 在 `BattleRedraw` 之后执行。背景 Blt 到 DD backbuffer，文字画到 `screen_pcx16`。HD_TC2 后续把 `screen_pcx16`（含文字）翻转到 backbuffer（含背景），最终两者叠加显示。

## 双缓冲同步

因为 `screen_pcx16` 不会被自动清除面板区域，旧文字会残留。必须每帧画面板背景时同步清除 `screen_pcx16` 对应区域，否则会出现"两层数值"。

当前实现：

1. Blt 面板背景到 DD backbuffer。
2. 清除 `screen_pcx16` 面板区域的旧像素（填 0）。
3. 画新文字到 `screen_pcx16`。

相关函数：

| 函数 | 用途 |
|---|---|
| `ClearScreenRegion()` | 清除 `screen_pcx16` 面板区域旧像素 |

## PCX 背景加载

当前方案完全绕开 `_Pcx8_/_Pcx16_` 图像绘制，直接使用 DirectDraw：

```text
PCX 文件
  -> 手动解码 1-plane 8-bit PCX + 调色板
  -> o_DD->CreateSurface() 创建离屏 surface
  -> Lock surface 写入 RGB565 或 32-bit 像素
  -> o_DDSurfaceBackBuffer->Blt() 画到 backbuffer
```

相关全局变量：

| 全局变量 | 地址 | 类型 |
|---|---:|---|
| `o_DD` | `0x6AAD20` | `LPDIRECTDRAW` |
| `o_DDSurfacePrimary` | `0x6AAD24` | 主 surface |
| `o_DDSurfaceBackBuffer` | `0x6AAD28` | backbuffer |

## PCX 支持范围

当前只支持：

- 1-plane
- 8-bit indexed color
- 文件尾含 768 字节调色板

配置项：

```ini
[RangedPanel]
BackgroundImage=rp_bg.pcx
```

图片放在插件目录的 `img` 子目录。

## 面板绘制流程

完整流程：

1. `CreateDDBackground()` 读取并解码 PCX 到离屏 DD surface。
2. `SaveUnderlay()`：从 backbuffer 保存面板区域原始像素到离屏 surface。
3. `DrawDDBackground()`：背景 Blt 到 DD backbuffer。
4. `ClearScreenRegion()`：清除 `screen_pcx16` 面板区域旧文字像素。
5. `Recalculate()`：按需重算数值（dirty/checksum 触发）。
6. `SafeDrawText()`：文字画到 `screen_pcx16`。

## 面板隐藏与残留处理

面板位于 800x600 战场框外部上方。快速战斗结果界面或战斗结果界面不会可靠覆盖这一区域。

残留分两类：

| 残留类型 | 来源 | 处理方式 |
|---|---|---|
| 背景残留 | DD backbuffer 上的面板背景 | underlay 保存/恢复 |
| 文字残留 | `screen_pcx16` 上的旧文字 | 每帧 `ClearScreenRegion` |

### DD backbuffer 残留：underlay 备份

1. 首次画面板前，把面板区域下方的 backbuffer 原始像素复制到离屏 surface。
2. 只保存一次（坐标不变时跳过），避免把面板自身保存进去。
3. suppress/destroy 时，把保存的原始像素 Blt 回 backbuffer。
4. 坐标变化时，先恢复旧 underlay，再保存新位置。

相关函数：

| 函数 | 用途 |
|---|---|
| `CreateEmptyDDSurface()` | 创建空离屏 surface |
| `CopyBackBufferToSurface()` | 从 backbuffer 保存面板底下原始像素 |
| `RestoreSurfaceToBackBuffer()` | 把保存的原始像素恢复到 backbuffer |
| `SaveUnderlay()` | 面板绘制前保存 underlay |
| `RestoreUnderlay()` | 面板隐藏/销毁时恢复 underlay |

### screen_pcx16 残留：每帧清除

每帧画面板背景后、画文字前，用 `ClearScreenRegion()` 将 `screen_pcx16` 的面板区域填 0。确保旧文字不留到下一帧。

## 不工作的方案

| 方案 | 结果 | 原因 |
|---|---|---|
| 自定义 `_Pcx8_::DrawToPcx16` | 崩溃 | HD_TC2 hook 访问自定义 Pcx8 缺失的内部状态 |
| 注册到 `o_BinTreeHead` 资源树 | 崩溃 | HD_TC2 检查的不只是资源树归属 |
| 手写 `screen_pcx16->buffer` 作背景 | 显示错乱 | HD_TC2 使用自己的最终渲染管线 |
| `_Pcx16_::DrawPcx16` 风格手动写 buffer | 显示错乱 | 本质仍是写逻辑 buffer |
| `DDBLT_COLORFILL` 填黑清除 | 出现黑块 | 面板区域在结果界面覆盖范围外 |
| suppress 后调用 `RedrawBattlefield` | 不可靠 | 面板在战场框外部，战场重绘不保证覆盖该区域 |
| 不清除 `screen_pcx16` 旧文字 | 两层数值 | `screen_pcx16` 不自动清除面板区域 |

## 注意事项

- `#include <ddraw.h>` 必须在 `homm3.h/H3API.hpp` 相关头之前可用。
- DirectDraw 调用都应容错，失败只影响面板显示，不应带崩游戏。
- underlay 必须在面板首次绘制前保存，不能每帧重复覆盖。
- 坐标或尺寸变化时，应先恢复旧 underlay，再保存新 underlay。
- `ClearScreenRegion` 每帧调用，必须在画文字之前执行。
- `screen_pcx16` 的像素格式假设为 16-bit RGB565。
