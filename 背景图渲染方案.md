# 战场远程对比框背景图渲染方案

## 背景

H3 BattleValueInfo 插件需要在战场顶部显示一个远程/法术对比面板（298×93 像素），背景图从 PCX 文件加载。

## 问题

最初使用 DirectDraw 的 `Lock()` + 直接写入像素 + `Unlock()` 方式，在原版 SoD 可行，但在 HD 版（使用 OpenGL 渲染层 HD_VGL2.dll）中：
- backbuffer 的像素格式是 BGRA（A=0，即完全透明）
- 直接写入像素后颜色完全不对
- 尝试 `Blt()` 到 backbuffer 会触发 HD_VGL2 崩溃（HD 版的 backbuffer 由 OpenGL 层管理）

## 解决方案

**完全不使用 DirectDraw**，所有绘制都走游戏原生 `screen_pcx16`（16bpp RGB565 surface）。

### 核心流程

```
PCX 文件 (8bpp indexed color)
    ↓ RLE 解码 + palette 查表
_Pcx16_ 对象 (RGB565 格式，bg_)
    ↓ bg_->DrawToPcx16(x, y, FALSE, screen)
screen_pcx16 (游戏主屏幕 surface)
    ↓ 游戏自动上传到 GPU
屏幕显示
```

### 关键点

1. **创建背景**：`CreatePcx16Background()` 解码 PCX，生成 RGB565 格式的 `_Pcx16_` 对象
2. **清除区域**：每帧先清除面板区域（避免残留）
3. **画背景**：`bg_->DrawToPcx16(x, y, FALSE, screen)` — 使用 H3API 的 `H3LoadedPcx16::DrawToPcx16`，透明参数为 FALSE（不透明覆盖）
4. **画文字**：`font->TextDraw(screen, text, x, y, w, h, color, alignment)` — 直接画到 screen_pcx16

### 代码示例

```cpp
// 确保背景已加载
if (!bg_) {
    bg_ = CreatePcx16Background(path);
    bg_load_failed_ = (bg_ == nullptr);
}
if (!bg_) return;

// 1. 清除面板区域
for (int row = y; row < y + panel_h; ++row) {
    _word_* p = (_word_*)(screen->buffer + row * screen->scanlineSize) + x;
    for (int col = 0; col < panel_w; ++col) p[col] = 0;
}

// 2. 画背景到 screen_pcx16
bg_->DrawToPcx16(x, y, FALSE, screen);

// 3. 画文字
_Fnt_* font = GetRangedPanelTextFont();
if (font) {
    font->TextDraw(screen, text, x, y, w, h,
        NH3Dlg::eTextColor::RED,
        static_cast<NH3Dlg::eTextAlignment>(4)); // MIDDLE_LEFT
}
```

## 注意事项

- **HD 版必须用 screen_pcx16**：HD 版的 backbuffer 由 OpenGL 管理，不能直接 Blt/Lock
- **清除区域防止残留**：每帧必须清除面板区域，否则上一帧的文字会残留
- **Hook 位置**：使用 `Hook_CycleCombatScreen` (0x495C50)，在每帧渲染完成后触发
- **RGB565 格式**：screen_pcx16 是 16bpp RGB565，背景解码时必须用 `RGB565_fromR8G8B8()` 转换

## Hook 地址参考

| 名称 | 地址 | 说明 |
|------|------|------|
| Hook_CycleCombatScreen | 0x495C50 | 战斗动画循环，每帧调用，在 backbuffer 渲染完成后 |
| Hook_AfterBlt | 0x600430 | 在 DD Blt 操作后，但在 Clear 之前（不推荐） |

## 历史

- 最初用 DD Lock/Write，原版正常，HD 版颜色不对
- 尝试 Blt 到 backbuffer，HD 版崩溃
- 最终方案：完全不用 DD，所有绘制走 screen_pcx16
